처음 방문한 사용자가 "무엇을, 왜, 어떻게" 만드는지 빠르게 이해할 수 있도록 소개 문구와 구조를 재정리합니다.

- 프로젝트 첫 10초 이해: 아버님을 위한 실시간 주식 모니터링 + 자동매매 확장 프로젝트임을 명확히 전달
- 보안 인식 강화: API 키/계좌정보 관리 원칙을 초기 구간에서 강조
- 실행 가능성 제시: 단계별 개발 로드맵(8단계)을 유지하되 읽기 쉽게 개선

- 민감정보(.env, .gitignore) 보안 가이드가 유지된다.
- 기존 개발 흐름(증권사 선정 -> 인증 -> 실시간 수신 -> 저장/시각화 -> 전략/주문 -> 운영 안정화)이 훼손되지 않는다.
