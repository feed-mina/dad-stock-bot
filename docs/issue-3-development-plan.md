# Issue #3 세부 개발 플랜

대상 이슈: <https://github.com/feed-mina/dad-stock-bot/issues/3>

## 1. 외부 준비 작업

- 한국투자증권(KIS) 또는 키움 중 1개 API를 확정한다. 현재 코드 기준은 KIS Open API이다.
- 모의투자 계좌, AppKey, SecretKey를 발급한다.
- 민감정보는 `.env`에만 저장하고 Git에는 올리지 않는다.

## 2. 이번 브런치 구현 범위

- Python 패키지 구조와 `.env` 기반 설정을 추가한다.
- KIS OAuth 토큰 발급, 웹소켓 approval key 발급, 국내 주식 현재가 조회 클라이언트를 추가한다.
- 공공데이터포털 금융위원회 주식시세정보 기반 일별/지연 시세 Provider를 추가한다.
- 국내 주식 실시간 체결가 `H0STCNT0` 구독 메시지 생성과 수신 데이터 파서를 추가한다.
- SQLite 저장소에 REST 현재가와 웹소켓 체결 데이터를 누적 저장한다.
- 관심종목 일괄 동기화, 최근 저장 데이터 조회, CSV 내보내기를 추가한다.
- 같은 종목/기준일의 공공데이터 중복 저장을 방지하고 기존 중복 정리 명령을 추가한다.
- 최소 전략으로 이동평균 돌파 신호를 만든다. 이번 단계에서는 실제 주문을 전송하지 않는다.
- 텔레그램 알림은 신호 발생 시 보낼 수 있는 선택 기능으로 둔다.

## 3. 후속 단계

- 공공데이터포털 인증키로 `daily-quote` 명령을 검증한다.
- 공공데이터포털 인증키로 `daily-sync`를 실행해 관심종목 전체를 SQLite에 저장한다.
- `dedupe`를 실행해 기존 중복 저장 행을 정리한다.
- `latest`, `export-csv`로 저장 데이터 조회와 스프레드시트용 파일 출력을 검증한다.
- 모의투자 계정으로 KIS REST `quote` 명령을 검증한다.
- 장중에 `listen` 명령으로 웹소켓 수신, 재연결, 저장을 검증한다.
- 엑셀 또는 Google Sheets 연동 방식을 선택한다.
- 모의투자 주문 API 어댑터를 추가하고, 주문 전송 전 승인/한도/중복 주문 방지 규칙을 둔다.
- 운영 로그, 장애 대응, 토큰 만료 자동 복구 시나리오를 문서화한다.

## 4. 완료 기준

- `check-config`가 비밀값 없이 설정 요약을 출력한다.
- API 키가 있으면 REST 현재가가 조회되고 SQLite에 저장된다.
- 장중 웹소켓 연결 시 체결 데이터가 파싱되어 SQLite에 저장된다.
- 단위 테스트가 키와 네트워크 없이 통과한다.
